By passing the float prop to the component, we can have a nice label which floats 
above the handle and shows the current value.

<Info>Floats can be formatted separately to the pips by using [custom `handleFormatter` functions](../options#handleFormatter).</Info>

<Demo name="Floating Labels" client:visible>
  <Float client:visible />
  <div slot="svelte">
    ```svelte
    <RangeSlider float />
    ```
  </div>
</Demo>

<Note type="info">
  I've manually set the above Slider to be focussed on load for this demo, so that the float is
  visible and obvious. [See below for how to do this](#focus-on-load).
</Note>



## Float without Hover

The Slider can be initialised with `hoverable` set `false` which will prevent
the float from appearing on hover.

<Demo name="Floating only on focus" client:visible>
  <NoHover client:visible />
  <div slot="svelte">
    ```svelte
    <RangeSlider float />
    <RangeSlider float hoverable={false} />
    ```
  </div>
</Demo>

<Note type="warning">
  This will also **prevent all other hover effects** on the Slider. If you wish
  only to hide the hover effect for floats, you may edit the CSS for more control.
</Note>



## Floats on the Range

In version `4.0.0` we added the ability to have a float on the range _(area between handles)_. This is useful for
when you want to show the current value of the range.

<Info>Hover/Drag the Slider handles below to see the float</Info>

<Demo name="Floating Labels" client:visible>
  <RangeFloat client:visible />
  <div slot="svelte">
    ```svelte
    <RangeSlider rangeFloat float />
    ```
  </div>
</Demo>

<Note type="info">
  It would be possible to toggle the `float`/`rangeFloat` properties based on the distance between the handles.
  [See an example below.](#automatically-toggle-rangefloat)
</Note>



## Always show the floats

We can set the floats to always be visible by modifying the css slightly. This can either
be done in an external css file or inline in the Svelte component.

<Demo name="Always Shown Floats" client:visible panel="css">
  <Always client:visible />
  <Code slot="svelte" code={cleanImport(AlwaysRaw)} lang="svelte" />
  <Code slot="css" lang="css" code={`/* styling in an external css file */
#always .rangeFloat {
  opacity: 1;
  translate: -50% 0%;
}`} lang="css" />
</Demo>

## Automatically toggle `rangeFloat`

Here we demonstrate a simple example of how to toggle the `rangeFloat` and `float` properties 
based on the distance between the handles.

<Demo name="Toggle rangeFloat and float" client:visible>
  <ToggleRangeFloat client:visible />
  <Code slot="svelte" code={cleanImport(ToggleRangeFloatRaw)} lang="svelte" />
  <Code slot="css" lang="css" code={ToggleRangeFloatCss} />
</Demo>

This simple example is using the `values` property to toggle the `rangeFloat` and `float` properties.
But it would be possible to use the `on:change` event, and calculate the pixel-distance between the handles,
if that was a better approach for your use case.


## Keep Float inside container

If your slider has **very long float labels**—and is positioned close to the edge 
of the browser or its parent container—you may find the floats extend past the
boundaries and look terrible.

One way to **keep floats inside is by using a simple css `calc()` to re-position the
floats *based on the handle's position***.
